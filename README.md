
# SaleFlow POS - ระบบจัดการจุดขายร้านอาหารสัตว์

ยินดีด้วย! แอปพลิเคชัน SaleFlow POS ของคุณพร้อมใช้งานแล้ว นี่คือคำแนะนำสำหรับการนำไปใช้งานจริงและการจัดการข้อมูล

## 📊 การจัดการข้อมูล (Data Management)
ข้อมูลทั้งหมดของระบบถูกจัดเก็บไว้ใน **Firebase Cloud Firestore** อย่างปลอดภัย
- **วิธีดูข้อมูล**: เข้าไปที่ [Firebase Console](https://console.firebase.google.com/)
- **เมนู**: เลือกโปรเจกต์ของคุณ -> **Build (สร้าง)** -> **Firestore Database**
- **สิ่งที่คุณทำได้**: ตรวจสอบรายการขายทั้งหมด, แก้ไขสต็อกสินค้าโดยตรง, หรือลบข้อมูลพนักงานในกรณีฉุกเฉิน

## 🚀 การนำไปใช้งานจริง (Deployment)
เพื่อให้พนักงานเข้าใช้งานได้จากมือถือหรือคอมพิวเตอร์ส่วนตัว คุณต้องทำการ Deploy แอปพลิเคชันผ่าน **Firebase App Hosting**:

1. **เชื่อมต่อ Repository**: นำโค้ดโปรเจกต์ของคุณขึ้น GitHub หรือ GitLab
2. **สร้าง Backend**: ใน Firebase Console ไปที่เมนู **App Hosting** แล้วกด **Get started**
3. **กำหนดการตั้งค่า**:
   - เลือก Repository ที่ต้องการ
   - ระบุ Branch (เช่น `main`)
   - ระบบจะใช้ไฟล์ `apphosting.yaml` ที่เตรียมไว้ในการกำหนดค่า Build อัตโนมัติ
4. **URL ใช้งาน**: เมื่อ Deploy สำเร็จ คุณจะได้รับ URL ถาวร (เช่น `https://saleflow-pos.web.app`) เพื่อนำไปให้ทีมงานใช้งาน

## 📱 การใช้งานบนมือถือ
เมื่อ Deploy สำเร็จแล้ว ให้พนักงานเปิด URL ผ่านเบราว์เซอร์บนมือถือ (Safari/Chrome) และแนะนำให้กด **"เพิ่มลงในหน้าจอโฮม" (Add to Home Screen)** เพื่อให้ใช้งานได้สะดวกเหมือนแอปพลิเคชันทั่วไป (PWA)

## 🔐 ความปลอดภัย
- อย่าลืมตั้งค่า **Firebase Authentication** ในหน้าคอนโซลให้เปิดใช้งานวิธี Login แบบ Email/Password
- ตรวจสอบสิทธิ์พนักงานในเมนู "จัดการพนักงาน" ภายในแอปเป็นประจำ
