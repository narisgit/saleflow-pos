
# SaleFlow POS - คู่มือสำหรับนักพัฒนา (Developer Guide)

โปรเจกต์นี้เป็นระบบจุดขาย (POS) ที่สร้างด้วย Next.js และ Firebase โดยใช้ GitHub Actions ในการ Deploy อัตโนมัติ

## 🚀 วิธีออนไลน์ระบบแบบฟรี (GitHub Actions + Firebase Hosting)

### การตั้งค่าครั้งแรก (ทำครั้งเดียว)
1. **Firebase Console**: ไปที่ Project Settings > Service Accounts > Generate new private key เพื่อโหลดไฟล์ `.json`
2. **GitHub Settings**: ไปที่ Repository > Settings > Secrets > Actions
3. **Add Secret**: สร้าง Secret ชื่อ `FIREBASE_SERVICE_ACCOUNT` แล้ววางเนื้อหาจากไฟล์ `.json` ลงไป
4. **Push Code**: เมื่อคุณ Commit & Push โค้ดจาก Firebase Studio ระบบจะเริ่ม Deploy ทันที

## 🛠 วิธีตรวจสอบว่าหน้าเว็บอัปเดตหรือยัง?
หากคุณกด Push แล้วแต่หน้าเว็บยังเป็นแบบเดิม ให้ตรวจสอบตามลำดับดังนี้:

1. **เช็กที่ GitHub Actions**:
   - ไปที่หน้า Repository ใน GitHub ของคุณ
   - กดเมนู **Actions** ด้านบน
   - หากเห็นวงกลมสีเหลืองหมุนอยู่ แปลว่าหุ่นยนต์กำลังทำงาน ให้รอจนกว่าจะเป็น **เครื่องหมายถูกสีเขียว**
2. **เช็กตัวระบุเวอร์ชันในแอป**:
   - ผมได้เพิ่มแถบสีเขียวเล็กๆ ที่เมนูด้านซ้าย (Sidebar) เขียนว่า **"Updated at [เวลา]"**
   - หากเวลาที่แสดงยังเป็นเวลาเก่า แปลว่าหน้าเว็บในมือถือคุณยังไม่ได้อัปเดต
3. **วิธีบังคับอัปเดต (Clear Cache)**:
   - **บนมือถือ**: ให้ปิด Browser ทุกแท็บ แล้วเปิดลิงก์ใหม่ หรือเปิดใน "โหมดไม่ระบุตัวตน"
   - **บนคอมพิวเตอร์**: กด `Ctrl + F5` เพื่อรีโหลดหน้าเว็บใหม่ทั้งหมด

## 📱 วิธีหาลิงก์เข้าใช้งาน (App URL)
1. ไปที่ **[Firebase Console](https://console.firebase.com/)**
2. เลือกโปรเจกต์ของคุณ -> เมนู **Hosting** ที่แถบด้านซ้าย
3. ลิงก์จะอยู่ในช่อง **Domains** (เช่น `https://your-app.web.app`)

## 👥 ขั้นตอนการเพิ่มพนักงานเข้าสู่ระบบ
1. ให้พนักงานสมัครสมาชิกเองที่หน้า **Register** ในแอป
2. เมื่อสมัครเสร็จ Admin จะเห็นรายชื่อในหน้า **"จัดการพนักงาน"**
3. Admin กดแก้ไข (ไอคอนดินสอ) เพื่อปรับตำแหน่งเป็น **Manager** หรือ **Admin** และตั้งรหัสพนักงานได้ตามต้องการ
